1.5.2　挨拶文変更コマンド

ICommandインターフェース

public interface ICommand
{
    // コマンドを実行するためのメソッド
    void Execute(object? parameter);

    // コマンドが実行可能かどうかを調べるメソッド
    bool CanExecute(object? parameter);

    // コマンドの実行可能・不可能が変化したら呼び出すべきイベント
    event EventHandler? CanExecuteChanged;
}

ViewModel.cs

using Prism.Commands;

namespace SampleCode
{
    internal class ViewModel
    {
        public ViewModel()
        {
            ChangeMessageCommand = new DelegateCommand(() => GreetingMessage = "Bye-bye world");
        }
        public string GreetingMessage { get; set; } = "Hellow World!";

        public DelegateCommand ChangeMessageCommand { get; }
    }
}

INotifyPropertyChanged

public interface INotifyPropertyChanged
{
    event PropertyChangedEventHandler? PropertyChanged;

ViewModel.cs

using Prism.Commands;
using System.ComponentModel;

namespace SampleCode
{
    internal class ViewModel:INotifyPropertyChanged
    {
        public ViewModel()
        {
            ChangeMessageCommand = new DelegateCommand(() => GreetingMessage = "Bye-bye world");
        }
        private string _greetingMessage = "Hellow World!";
        public string GreetingMessage {
            get =>_greetingMessage;
            set
            {
                if(_greetingMessage != value)
                {
                    _greetingMessage = value;
                    PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(nameof(GreetingMessage)));
                }
            }
        }

        public DelegateCommand ChangeMessageCommand { get; }

        public event PropertyChangedEventHandler? PropertyChanged;
    }
}

ViewModel.cs

        }

        public string GreetingMessage
        {
            get => _greetingMessage;
            set => SetProperty(ref _greetingMessage, value);

8章 Xaml用語集

　本章では、よく使われるXamlの用語について説明します。

要素

　要素名は、C#のクラス名です。Xaml中に要素を書くと、WPFは、そのクラスのオブジェクトを作ります。

　要素により作られるオブジェクトは、画面に表示されるコントロール、整数や文字列、表示する際に利用されるデータなど様々です。

子要素

　子要素は、要素の子として配置された要素のことです。WPFは、子要素に対しても、要素名と同じ名前のクラスのオブジェクトを作ります。

　子要素のオブジェクトは、親要素のいずれかのプロパティに設定されます。そのプロパティは、親要素の種類毎に決まっており、上図の例では、Grid要素ではChildrenプロパティに、Button要素ではContentプロパティに設定されます。

　なお、後述するプロパティ要素は、子要素ではありません。

親要素

　子要素を含む要素です。

属性

　属性は、要素によって作られたオブジェクトが公開しているプロパティです。

属性値

　属性値は、プロパティに代入する値です。常に文字列で書かれますが、プロパティに代入するときに適切な型に自動的に変換されます。自動的に変換できない場合にはビルドエラーを起こします。

　属性値は、単純な文字列で書ける程度の値しかプロパティに代入できません。複雑な値をプロパティに代入するには、プロパティ要素を使う必要があります。

プロパティ要素

　プロパティ要素は、属性と同じく、要素によって作られたオブジェクトが公開しているプロパティです。単純な文字列で書けないほど複雑な値をプロパティに設定するために使われます。値の複雑さによって、その値を属性に書くかプロパティ要素に書くかを決めてください。

　なお、プロパティ要素は、コンテントではありません。

依存関係プロパティ

　依存関係プロパティは、クラスのプロパティの一種ですが、普通の値に加えてバインディングやリソースを代入できます。

　Xamlからは、普通のプロパティと同じように属性やプロパティ要素により設定できます。C#からは、普通のプロパティと同じように読み書きできます。

　プロパティが依存関係プロパティの場合、マイクロソフトのドキュメントには、プロパティの説明文の最後に「依存プロパティ情報」（英語版では「Dependency Property Information」）が記載されています。

コンテント

　要素のコンテントとして書かれる文字列や要素です。上図では、Grid要素内のTextBlock要素とButton要素、そして、Button要素内の文字列「Change message」がコンテントです。

　コンテントは、親要素のプロパティに代入されます。コンテントを格納するプロパティを、コンテントプロパティと呼びます。どのプロパティがコンテントプロパティかは、親要素の種類により異なります。例えば、Grid要素ではChildrenプロパティが、Button要素ではContentプロパティがコンテントプロパティです。

添付プロパティ

　添付プロパティは、それが置かれている要素の属性ではありません。他の要素が用意した粘着メモのようなものです。

　この粘着メモには、それを用意した要素のためのメモが書かれます。図の例では、Grid要素が用意した行番号用粘着メモに「1」と書いて、「Button要素をGridの2行目に配置してほしい」と伝えています。

マークアップ拡張

　マークアップ拡張は、普通に書くと複雑で読みにくくなる値を読みやすく書けるように工夫された書き方です。

　{Binding ...}は、依存関係プロパティとして作られた属性を、任意の要素の属性や、DataContextに代入されているオブジェクトのプロパティに関連付けます。

　{x:Type className}は、C#でType.GetType("className")と書いたのと同じ意味です。属性にはType型オブジェクトが代入されます。

　{StaticResource keyName}は、リソースディクショナリに登録されているkeyNameというキーを持つエントリーの値を参照します。リソースディクショナリの値が入れ替えられても、それを検出せず、古い値、つまり、それまで参照していたオブジェクトを参照し続けます。

　{DynamicResource keyName}は、リソースディクショナリに登録されているkeyNameというキーを持つエントリーの値を参照します。リソースディクショナリに登録されている値が入れ替えられると、それを検出して、新しい値を参照します。

　{x:Null}は、C#のnullです。
